This maps the movement of Matter.js bodies to the movement of HTML Elements! Pretty much any attribute can be stored on the body reference, get altered in the Javascript, and be applied when rendered with the requestAnimationFrame function.

The regular Matter.js canvas is overlayed onto the HTML with 0 opacity so it can recieve mouse events. Unfortunately this did not help me with my original goal, as I wanted to have 'a' tags inside the mapped elements.

For now, the 'link' function is being handled by a mouseUp event which is not ideal because if you mouse down off of a link, then mouse over a link that has a mouseUp event, it still gets triggered... Not exactly normal link behavior.

It allows you to render text and images though, so still pretty neat!
